# Shady Garage & Speed — iOS

A fully native iOS port of the web game **Shady Garage & Speed** (`/Users/ldr/shady-garage-speed`).
SwiftUI + SceneKit, zero dependencies, zero binary assets — every model, texture, and
sound is generated procedurally in code, exactly like the web version.

Fix cars. Maybe steal parts. Build a racer.

## Requirements

- Xcode 26 (developed on 26.6)
- iPhone simulator or device running iOS 17.0+ (iPhone-first, portrait + landscape)
- No packages, no CocoaPods, no network dependencies

## Run

1. Open `ShadyGarageSpeed.xcodeproj` in Xcode.
2. Select an iPhone simulator (e.g. iPhone 16 Pro).
3. Press **Cmd+R**.

Command line equivalent:

```sh
xcodebuild -scheme ShadyGarageSpeed \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro' \
  -configuration Debug CODE_SIGNING_ALLOWED=NO build
```

The Xcode project is generated by [XcodeGen](https://github.com/yonsm/XcodeGen) from
`project.yml`. If you add/remove source files, either re-run `xcodegen` or just let
Xcode know — all sources live under `ShadyGarageSpeed/` and are globbed by XcodeGen.
(If xcodegen is unavailable, the checked-in `.xcodeproj` works as-is.)

## How to play

- Customers pull into the bay. **Tap a part on the car** (or use the job panel) to inspect it.
- **Fix** worn parts for safe cash, or **Steal** them for yourself via the timing minigame —
  watch the **Suspicion** meter (at 100 the customer storms off) and the **Heat** meter
  (cops visit at ≥70, raid at 100).
- Spend cash and stolen parts in the **Build Bay** (chassis upgrades, 6 part slots).
- Prove it in the **time trial**: day/sunset/night conditions cycle per race, 35% rain chance,
  NOS boost, best-lap tracking, rival leaderboard.

## Feature map (web → iOS)

| Web (`js/*.js`) | iOS (`ShadyGarageSpeed/`) |
| --- | --- |
| `data.js` — state, tiers, perks, customers, stats, save | `GameState.swift` — `ObservableObject`, identical formulas/constants, JSON in `UserDefaults` (`sgs_save`) with backfill migration |
| `audio.js` — WebAudio oscillators | `AudioEngine.swift` — AVAudioEngine with pre-rendered mono PCM buffers (same freqs/durations/envelopes/slides + NOS loop) |
| `cars.js` — procedural cars/avatars | `SceneKit/CarFactory.swift` — same dims/positions; named part nodes; tier tints; chassis L1–L4 visuals |
| `garage.js` — scene, customer loop, raycast | `SceneKit/GarageScene.swift` — same layout/constants; tap hit-testing; drive tweens; suspicion/rage; heat/cop/raid |
| `build.js` — build bay | `SceneKit/BuildScene.swift` — lift + turntable + identical props/actions |
| `race.js` — track, physics, conditions | `SceneKit/RaceScene.swift` — closed Catmull-Rom (τ=0.6, 15 control points, 800 arc-length samples), identical arcade physics, lap logic, NOS, day/night + rain, chase cam, minimap data |
| `ui.js` + `index.html` — DOM HUD | `Views/*.swift` — SwiftUI HUD (menu, setup, garage, build, race, results, steal minigame, toasts, cop modal) |
| `main.js` — phase state machine | `AppState.swift` + `GamePhase.swift` |

All tuning constants (stat formulas, prices, suspicion/heat numbers, race physics,
rival times, NOS drain/regen, grip factors, camera curves) are copied from the web
source verbatim.

## Native adaptations

- **UIKit haptics**: steal success/fail, cop visit/raid, lap finish.
- **Tap instead of hover**: parts highlight on tap (cyan emissive pulse) and the matching
  job-panel row highlights.
- **Portrait camera framing**: in portrait the garage/build cameras tilt slightly so the car
  clears the bottom-sheet panels (web uses the same 45vh bottom sheets on phones).
- Layout mirrors the web's responsive CSS: bottom sheets on portrait phones, side panels
  in landscape/wide, compact race HUD (96pt minimap, 64pt buttons) on small screens.
- Sign uses `SCNText` (the web uses a hand-built voxel font); hemisphere lights map to
  SceneKit ambient lights; instanced meshes map to `flattenedClone()` node clones;
  rain uses the same recycled-node approach as the web.

## Debug launch args

Jump straight into a phase (used for headless screenshots/testing):

```sh
xcrun simctl launch booted com.amurchadha.shadygaragespeed -phase race -tod night -rain off -autodrive
```

- `-phase menu|setup|garage|build|race` — start in a phase.
- `-tod day|sunset|night` — force race time-of-day (overrides the raceCount cycle).
- `-rain on|off` — force race weather.
- `-autodrive` — race drives itself (holds gas, steers to the centerline); handy for mid-race screenshots.

## Notes

- **Race start fix**: the road ribbon (the only custom `SCNGeometry` in the app) originally
  built its `SCNGeometryElement` from an `[Int]` index array — 8-byte indices are invalid
  (`bytesPerIndex` supports 1/2/4), so SceneKit silently never drew the road surface and the
  car appeared to sit on grass. The mesh now uses an explicit `UInt32` data-based element.
- The garage/build cameras tilt slightly in portrait so the car clears the bottom sheets.

## Persistence

Game state autosaves to `UserDefaults` (JSON, key `sgs_save`) on every meaningful action —
same fields as the web `localStorage` save, with the same `applyLoaded`-style backfill so
older saves missing new fields still load. (Web and iOS saves are not cross-compatible.)
